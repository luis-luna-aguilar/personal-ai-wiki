AI Wiki

A personal knowledge base for tracking the fast-moving AI landscape, maintained by an LLM.

This manual is for you, the human. It explains the workflows you use to drive the wiki. It does not cover how the LLM maintains pages internally — that's in CLAUDE.md.

What this is

This vault is a personal AI-tracking wiki. You drop in sources (articles, tweets, papers, podcast transcripts, GitHub repos, newsletters, meeting notes, your own notes). The LLM reads them, proposes diffs to the wiki, waits for your approval, and applies what you accept. Over time, the wiki becomes a structured, cross-referenced, always-current map of the AI landscape as you understand it.

The core loop

  1. Drop a source into raw/ — or give the LLM a URL.
  2. Say "ingest this" (or "triage this" for newsletters).
  3. Review the proposal in proposals/ — tick the boxes for changes you want.
  4. Say "apply it" — the LLM writes only the checked changes.
  5. Ask questions any time. Answers come with freshness-based confidence.

Why it works this way

  • Current state first. Pages are present-tense. History is capped and mostly off-page. You don't get token-bloated by things that used to be true.
  • Dates are load-bearing. Every claim has an as_of date. Confidence is computed from age at query time — no credibility scores, no per-claim bookkeeping during ingest.
  • Dry-run everything. The LLM never writes to the wiki without your approval. Proposals are editable markdown files; you check boxes or edit drafts inline before applying.
  • Ambiguity is allowed. A subcategory can have multiple top players. "Cursor or Claude Code?" doesn't need a single winner.
  • Zero-cost hygiene. Maintenance is handled by Python scripts (no LLM tokens) — staleness, orphans, broken links, tag compliance. Semantic checks (contradictions, missing pages) run only when you explicitly ask.

Where things live

FolderPurpose
raw/Your source files, immutable. Subfolders by type.
wiki/LLM-maintained pages. The knowledge base.
personal/Your opinions and takes. Separate from the factual wiki.
proposals/Pending dry-run diffs waiting for your approval.
scripts/Python utilities for maintenance and URL fetching.
manual/This documentation. You're reading it now.
config.ymlThresholds and settings.
CLAUDE.mdOperating manual for the LLM (not for you).
