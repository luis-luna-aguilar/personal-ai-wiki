Config reference

config.yml at the vault root holds all tunable settings: staleness thresholds, history cap, scan paths, source-type playbooks, and URL-fetching behavior. Scripts read it directly. The LLM reads it when computing query-time confidence.

Where it lives

/config.yml (vault root, next to CLAUDE.md).

Sections

stale_thresholds_days

How many days a page can go without an as_of update before it's considered stale. Used by stale.py and by the LLM at query time to compute the confidence band.

stale_thresholds_days:
  state-of:  14
  model:      7
  tool:      30
  benchmark: 14
  workflow:  60
  concept:   90
  trend:     30
  source:   365    # sources don't really go stale; they're records
  default:   30    # fallback for unlisted types

Models and state-of pages have tight windows because they change the fastest. Concepts and workflows are more durable.

query.confidence_bands

How query answers are labeled. Ratios are multipliers of the page type's threshold.

query:
  confidence_bands:
    high_max_ratio:   1.0   # age <= 1.0x threshold  -> high
    medium_max_ratio: 2.0   # age <= 2.0x threshold  -> medium
    # otherwise                                      -> stale

The LLM uses the oldest relevant page's as_of when computing confidence. If multiple pages disagree on age, it reports the worst.

history.recent_changes_cap

How many entries a "Recent changes" section can hold on a main page before the oldest is spilled to wiki/history/. Default 5. Raise for more in-line trend visibility, lower for leaner pages.

history:
  recent_changes_cap: 5

stale.include_paths / stale.exclude_paths

Which directories stale.py scans. Everything under exclude_paths is skipped even if it would otherwise match include_paths.

stale:
  include_paths:
    - wiki/state-of
    - wiki/models
    - wiki/tools
    - wiki/benchmarks
    - wiki/workflows
    - wiki/concepts
    - wiki/trends
  exclude_paths:
    - wiki/history
    - wiki/sources
    - wiki/_schema
    - personal

Personal is excluded by default — your takes don't decay the same way facts do.

orphans.scan_paths / orphans.exempt_paths

Which pages orphans.py considers. Scan paths define what to check; exempt paths are never flagged as orphans.

orphans:
  scan_paths:
    - wiki
  exempt_paths:
    - wiki/index.md
    - wiki/log.md
    - wiki/state-of
    - wiki/_schema
    - wiki/history
    - wiki/sources

State-of pages are exempt because they're entry points — nothing links to them in the normal sense, they're the root of the graph.

source_types

Per-source-type configuration. The raw_dir determines where new files land; the ingest_style tells the LLM which playbook to follow; notes is a one-line reminder of what that playbook is.

source_types:
  article:
    raw_dir: raw/articles
    ingest_style: standard
    notes: "Full ingest. Extract key claims, update relevant pages."
  tweet:
    raw_dir: raw/tweets
    ingest_style: lightweight
    notes: "Often a signal only. May touch 1 page."
  paper:
    raw_dir: raw/papers
    ingest_style: deep
  podcast:
    raw_dir: raw/podcasts
    ingest_style: summarize_first
  repo:
    raw_dir: raw/repos
    ingest_style: readme_first
  newsletter:
    raw_dir: raw/newsletters
    ingest_style: triage
  meeting:
    raw_dir: raw/meetings
    ingest_style: personal
  note:
    raw_dir: raw/notes
    ingest_style: personal

Most edits you'd make here are just adding a new source type (e.g. youtube) and defining where it goes.

fetch_url

Settings for scripts/fetch_url.py.

fetch_url:
  profile_dir: ~/.cache/ai-wiki-playwright  # persistent Chromium profile for logins
  timeout: 30                               # page load timeout in seconds
  wait_for: networkidle                     # Playwright wait_until value
  download_images: true                     # download and rewrite image links to local paths
  images_dir: raw/assets                    # where downloaded images go

log_file

Where wiki/log.md lives. Rarely changed.

log_file: wiki/log.md

Changing thresholds

Start with the defaults, then adjust based on how often you actually ingest. Some heuristics:

  • If stale reports are constantly empty, tighten the thresholds.
  • If they're overwhelming, loosen the most noisy type first.
  • Models and state-of pages should always be the shortest — they change fastest.
  • Concepts and workflows can be generous — they change slowly.

Adding a new source type

  1. Add an entry under source_types with the raw_dir and ingest_style.
  2. Create the corresponding raw/<type>/ folder.
  3. Add an entry to wiki/_schema/source-types.md describing the playbook.
  4. Tell the LLM so it can use the new type on the next ingest.

Adding a new domain

  1. Ask the LLM to propose a new domain in a proposal file (vocabulary additions section).
  2. When you approve and apply, the LLM updates wiki/_schema/domains.md and creates a new wiki/state-of/<domain>.md skeleton.
  3. Optional: add a specific stale_thresholds_days entry if this domain's pages should decay differently. Otherwise they'll use default.
