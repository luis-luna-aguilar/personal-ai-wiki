Getting started

One-time setup: install Python dependencies for the scripts, configure Obsidian helpers, and know where to drop things.

1. Python dependencies

The maintenance scripts (stale.py, orphans.py, link_check.py, tag_compliance.py, lint_all.py) have no external dependencies — they run on stdlib Python 3.9+.

The URL fetcher (fetch_url.py) needs a real browser and a few libraries. Run this once:

pip install playwright markdownify readability-lxml
playwright install chromium

This installs Playwright (browser automation), markdownify (HTML→markdown), readability-lxml (article extraction), and a headless Chromium browser.

Why a real browser? Plain HTTP fetches get blocked by anti-bot systems on most news sites and block-free by design on Twitter, Substack, and paywalled content. A real Chromium instance bypasses nearly all of this and supports logins via persistent profiles.

2. Verify the scripts work

From the vault root:

python scripts/lint_all.py

You should see four sections (stale, orphans, link check, tag compliance) and an OK summary. On a fresh empty wiki, everything is green.

3. Obsidian setup

This vault is its own Obsidian vault — open the AI Wiki folder in Obsidian directly.

Recommended plugins / settings

  • Obsidian Web Clipper (browser extension): clip web articles directly to raw/articles/. When you clip, the LLM can then "ingest" the clipped file by name.
  • Attachment folder: Settings → Files and links → set Default location for new attachments to raw/assets/. This keeps all clipped images in one place.
  • Download attachments hotkey: Settings → Hotkeys → search "Download" → find Download attachments for current file and bind to Ctrl/Cmd + Shift + D. After clipping an article, hit the hotkey and all images download locally.
  • Graph view: the wiki structure becomes legible once you have a few pages — use it to spot orphans and hubs.

Or — skip the folder and just give the LLM a URL. It'll run fetch_url.py for you. See Fetching from URLs.

5. Your first source

Pick a recent article you'd like to capture. Drop it into raw/articles/ and tell the LLM:

ingest raw/articles/your-file.md

The LLM will draft a proposal file under proposals/. Open it in Obsidian, tick the boxes for what to apply, and say:

apply it

Repeat. That's the loop.

First ~10 sources will fan out aggressively. The wiki is empty, so new sources will create many new pages and subcategories. That's expected — after ~30 pages the LLM tightens up its reuse-first discipline automatically.
