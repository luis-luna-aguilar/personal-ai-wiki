Running scripts

Maintenance scripts in scripts/ are zero-token — they walk the vault's markdown files directly and produce reports. Run them whenever; they're cheap. The LLM can also run them on your behalf when you ask for a check.

When things run

Nothing runs automatically. No cron, no hooks, no post-commit triggers. Every script runs only when you (or the LLM, on your instruction) explicitly invoke it.

Suggested cadence:

  • Weekly: python scripts/lint_all.py before your review session. Catches most hygiene issues at once.
  • Ad-hoc: run stale.py when you want to know what needs refreshing, orphans.py when the graph view looks messy, link_check.py after a big refactor.
  • Semantic lint (LLM-based): monthly or before a major review. Explicit phrases only — see below.

The scripts

stale.py

Reports wiki pages whose as_of date has exceeded the staleness threshold for their type. Thresholds live in config.yml → stale_thresholds_days.

python scripts/stale.py            # human report
python scripts/stale.py --json     # machine-readable

Output groups pages into stale (age > 2× threshold), aging (age > threshold), and missing_as_of (frontmatter is incomplete).

Returns exit code 1 if any pages are stale, 0 otherwise.

orphans.py

Reports wiki pages with no inbound [[wikilinks]]. Exempt paths (state-of, _schema, history, sources, index, log) are never counted as orphans — see config.yml → orphans.

python scripts/orphans.py
python scripts/orphans.py --json

link_check.py

Reports broken [[wikilinks]] — targets that don't resolve to any file in wiki/.

python scripts/link_check.py
python scripts/link_check.py --json

tag_compliance.py

Reports any tag, domain, or subcategory used in a page's frontmatter that isn't declared in wiki/_schema/. Catches accidental typos and undeclared vocabulary.

python scripts/tag_compliance.py
python scripts/tag_compliance.py --json

lint_all.py

Chains all four scripts above into one combined report. This is the one you run weekly.

python scripts/lint_all.py

fetch_url.py

Real-browser URL fetch. See Fetching from URLs for details.

python scripts/fetch_url.py --type article --url https://...

Asking the LLM to run them

The LLM knows these trigger phrases:

You say…The LLM runs…
"show stale pages", "what's stale"stale.py
"find orphan pages"orphans.py
"check links"link_check.py
"check tag compliance"tag_compliance.py
"run lint", "lint the wiki"lint_all.py

Semantic lint (LLM-based, explicit only)

Some checks need semantic understanding and can't be scripted. The LLM runs these only when you explicitly ask, because they cost serious tokens. None are triggered automatically.

CheckTrigger phraseWhat it does
Contradictions "find contradictions", "scan for contradictions" Reads state-of + referenced pages, looks for claims that contradict each other.
Missing pages "missing page suggestions", "what concepts need pages" Finds entities/concepts mentioned across many pages but lacking their own page.
Domain completeness "domain completeness for X", "what's missing from state-of/X" Reports subcategories or topics that look thin compared to mentions elsewhere.
These will not run unless you ask with one of the trigger phrases above. Scripts are free; semantic lint is not. Treat it like a monthly review, not a routine check.

Exit codes

All scripts use these exit codes so they can be chained in CI or other automation:

  • 0 — no issues
  • 1 — issues found
  • 2 — script error (e.g., missing Playwright)
