Ingesting sources

Adding a source is a two-phase flow: drop the file, then review and apply the proposal. The LLM never writes to wiki/ directly.

The basic flow

  1. Drop a file into the right raw/ subfolder.
  2. Tell the LLM to ingest it. You'll get a proposal file in proposals/.
  3. Open the proposal in Obsidian, tick the checkboxes you agree with, optionally edit drafts.
  4. Tell the LLM to apply the proposal.

Source types and where they go

Each source type lives in its own raw/ subfolder and has a per-type ingest playbook the LLM follows. You don't need to remember the playbooks — just drop the file in the right folder.

TypeFolderWhat the LLM does
Article raw/articles/ Full ingest. Extracts claims, updates relevant pages, creates a source summary.
Tweet raw/tweets/ Lightweight. Often touches just one page. Preserves author + date.
Paper raw/papers/ Deep read. Extracts methodology, findings, limitations. May spawn concept pages.
Podcast raw/podcasts/ Summarizes the transcript first, then extracts claims. Preserves speaker attribution.
Repo raw/repos/ Reads README first. Becomes a tools/ entry.
Newsletter raw/newsletters/ Does not ingest directly. Always writes a triage file first. See Triaging newsletters.
Meeting notes raw/meetings/ Usually goes into personal/, not the factual wiki.
Your own notes raw/notes/ Usually goes into personal/.

Naming your files

Convention: YYYY-MM-DD-slug.md for most types. For repos, owner-repo.md (no date prefix). The LLM will rename files if needed, so this is a soft convention.

Examples

  • raw/articles/2026-04-09-cursor-3-release.md
  • raw/tweets/2026-04-08-karpathy-agent-loops.md
  • raw/papers/2026-03-28-voyage-rerank-2.md
  • raw/repos/anthropics-claude-code.md

What you say to the LLM

Any of these work:

  • "ingest raw/articles/2026-04-09-cursor-3-release.md"
  • "ingest the Cursor 3 article"
  • "ingest the latest thing in raw/articles"
  • "process raw/papers/voyage-rerank-2.md"

The LLM will read the file, check the index for existing pages it might touch, and draft a proposal. It will not modify the wiki until you say so.

What happens next

Once the proposal is ready, the LLM will tell you the path — something like proposals/2026-04-09-cursor-3-release.md. Open it in Obsidian.

See Proposals workflow for how to review and apply.

Batch ingesting several sources

You can drop several files at once and say "ingest all the new articles" — the LLM will process them sequentially and produce one proposal per source. Each is reviewed independently.

For newsletters with many signals, use the triage flow instead — don't try to ingest a newsletter as one article.

Clipped articles from the browser

Obsidian Web Clipper saves clean markdown straight into raw/articles/. After clipping, hit your image-download hotkey (Ctrl/Cmd + Shift + D) to pull images to raw/assets/. Then tell the LLM to ingest the file. For web content you don't want to clip manually, use URL fetching.
