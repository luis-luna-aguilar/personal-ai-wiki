**Microsoft drops three new AI models for speech, voice, and images:** The software giant just dropped its new [**MAI model family**](https://microsoft.ai/news/today-were-announcing-3-new-world-class-mai-models-available-in-foundry/) on Microsoft Foundry. MAI-Transcribe-1 is already topping speech-to-text benchmarks in 25 languages, beating out OpenAI’s Whisper and Google’s Gemini with speeds 2.5x faster than Azure’s current batch transcription. There's also MAI-Voice-1, which can clone a custom voice from just seconds of audio, and MAI-Image-2, which is now a top-three contender on Arena AI while generating images twice as fast.

**Anthropic cuts third-party tool support from Claude plans:** The AI lab just cut third-party tool support from its [**subscription plans**](https://x.com/bcherny/status/2040206440556826908), citing high demand that their current pricing couldn't handle. While tools like OpenClaw are out, you can still connect via discounted usage bundles or an API key, and existing subscribers are getting a one-time credit to cover the transition. OpenAI [**capitalized**](https://x.com/reach_vb/status/2040314906180735259) on the move immediately, pointing out that ChatGPT Plus still works seamlessly with OpenClaw, OpenCode, and Cline, while also promoting their open-source Codex App Server for custom builds.

**Karpathy turns LLMs into personal knowledge engines:** The OpenAI co-founder just pitched a new way to [**share projects**](https://x.com/i/trending/2040383163269100001): skip the code and share the concept, then let an AI agent build it for you. His first example uses LLMs to generate interconnected Markdown knowledge bases that are stored locally and viewable in tools like Obsidian. You own the data, the format is universal, and you can plug in any AI provider. Builders like Farza Majeed are already jumping [**on board**](https://x.com/karpathy/status/2040572272944324650), creating personal wikis from thousands of diary entries.

## **Tech's fastest-growing role has a trust problem**


--------------------

**AI just minted a new job title.** The forward-deployed engineer (FDE) barely existed outside Palantir until last year. Now, postings for the role have [**spiked**](https://www.salesforce.com/blog/forward-deployed-engineer/) 10x, with companies like Salesforce building out entire teams. FDEs embed with customers to make AI work in production, proving enterprise AI needs far more human hand-holding than expected.

**But engineers aren't biting.** The Wall Street Journal **[reports](https://www.wsj.com/cio-journal/the-hottest-job-in-tech-isnt-very-glamorous-dc29ab3e)** only 10% interest from talent, as many see it as a high-stress, low-prestige consulting gig. One dev even quit after four weeks, citing zero coding and too much client management.

**The title often overpromises.** While FDEs at OpenAI or Ramp build core products, most other roles are just sales support. Engineers usually spot this gap during the interview or shortly after starting.

**Ultimately, AI still needs a human touch.** Bridging the gap between a demo and production in messy enterprise environments requires writing code for legacy systems. Until these roles offer real engineering substance, the talent gap will only widen.

## **How to turn Figma designs into code with Codex**


--------------------
View image: (https://media.beehiiv.com/cdn-cgi/image/fit=scale-down,format=auto,onerror=redirect,quality=80/uploads/asset/file/b6f8eacd-7ed7-4bd3-8c6a-b2ba4b053d68/superhumanteam_a_software_engineer_sitting_at_a_desk_typing_o_40bc592f-a1db-4412-822e-ad95651f8673_1.jpg?t=1775473399)

Every frontend engineer knows the drill. A designer hands you a Figma file, and you spend the next few hours manually translating spacing, tokens, and layout into components. Codex can now do that [translation](https://developers.openai.com/codex/use-cases/figma-designs-to-code) for you using its Figma and Playwright skills.

Copy a link to the exact Figma frame you want built, then paste this prompt into Codex:

```
Implement this Figma design in the current project using the Figma skill.

Requirements:
- Start with `get_design_context` for the exact node or frame.
- Run `get_screenshot` for the exact variant before you start coding.
- Reuse the existing design system components and tokens.
- Make the page responsive on desktop and mobile.

Validation:
- Use Playwright to check that the UI matches the reference and iterate until it does.
```
Codex pulls design context from Figma’s MCP server and generates code that matches your repo’s patterns. It then uses Playwright to test the result in a live browser, iterating automatically until the UI perfectly matches the original design.

Both skills come built into Codex, so there's nothing to install.
