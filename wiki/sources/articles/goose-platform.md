# Goose platform overview

Goose's official docs position it as a local agent product rather than only a narrow framework library. The provider docs and repo together show a desktop, CLI, and API surface wrapped around provider configuration, MCP-related workflows, and local-agent execution.

## Influenced pages

- [Agentic orchestration patterns](../../workflows/agentic-orchestration-patterns.md) — adds a concrete example of the local agent host/product direction, contrasted with harness-first systems like Deep Agents

## Key claims extracted

- Goose supports a wide range of model providers, including OpenRouter
- The product exposes desktop and CLI setup flows and is documented as a broader user-facing agent surface
- Goose also supports MCP-related workflows through its docs and extension model
- This makes Goose a useful example of a local agent host/product layer, distinct from a batteries-included harness
