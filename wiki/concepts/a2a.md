# Agent2Agent

Agent2Agent, or A2A, is Google's protocol for communication between agents across systems. Where MCP is mainly about how an agent reaches tools and context, A2A is about how one agent delegates, coordinates, and exchanges structured work with another.

## Current status (as of 2026-04-22)

- Introduced by Google as an open interoperability protocol for agents
- Designed around enterprise-friendly network patterns rather than single-process agent loops
- Best understood as complementary to MCP, not a replacement for it

## Why it matters

- Gives multi-agent systems a cleaner interoperability story
- Makes cross-system delegation more legible than ad hoc API conventions
- Pushes the agent ecosystem toward protocol-based composition
