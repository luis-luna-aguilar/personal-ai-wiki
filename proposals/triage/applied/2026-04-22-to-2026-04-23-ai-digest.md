- [x] **[models, agents]** Anthropic 81K-person AI economics survey — workforce impact research with quantified findings ⚑ high-priority

    **What it is:** Anthropic published survey results from 81,000 Claude users on the economics of AI. Key quantified findings: (1) **Displacement concern tracks exposure** — people in high-AI-exposure jobs (software engineers, etc.) express job threat concern 3× more often than those in low-exposure occupations; early-career workers are significantly more worried than senior workers. (2) **Scope expansion is the primary benefit** — 48% of users report productivity gains via scope (doing new or previously impossible tasks); 40% via speed; quality and cost savings less common. Mean self-reported productivity rating: 5.1/7 ("substantially more productive"). (3) **U-shaped job threat relationship with speedup** — both those *slowed down* by AI (creative workers who find it stifling) AND those experiencing the *largest speedups* express heightened job displacement concern. The middle group (moderate speedup) is least worried. (4) **Benefits flow to workers, not employers** — most respondents say the gains go to themselves; only 10% say employers are extracting more work. High-wage and low-wage workers both report large gains; mid-tier scientific and legal workers are less enthusiastic. Early-career workers are less likely to personally benefit (60%) vs. senior workers (80%).

    **Why it matters:** This is Anthropic's own large-scale empirical evidence on AI's economic impact — the most direct self-sourced data the wiki has on this topic. The scope-expansion-over-speed finding contradicts the common "AI makes you faster" framing (it more often makes you *capable of new things*). The U-shaped threat relationship is a non-obvious finding worth tracking.

- [x] **[agents]** Microsoft Foundry Hosted Agents — production-grade per-session VM sandboxes for enterprise agents ⚑ high-priority

    **What it is:** Microsoft launched hosted agents in Foundry Agent Service (public preview). The key architectural distinction from traditional compute: every agent session gets its own **hypervisor-isolated VM sandbox** with a **persistent filesystem that survives scale-to-zero events** — not container sharing, not a code-execution-only sandbox. Features: predictable cold starts, scale-to-zero with resume intact, per-session isolation via isolation keys, BYO VNet, per-agent **Entra Agent ID** (not shared service accounts), built-in observability (OpenTelemetry), and weighted version rollouts. Framework-agnostic: supports LangGraph, Microsoft Agent Framework v1.0 (which unifies Semantic Kernel + AutoGen), Claude Agent SDK, OpenAI Agents SDK, GitHub Copilot SDK. Multi-model. Also shipping alongside: **Toolbox** (unified MCP-compatible tool management with OAuth passthrough), **Memory** (managed long-term memory native to Foundry), **Agent Registry** (central governance catalog), **Agent Gateway** (policy enforcement + Model Armor against prompt injection), and **Agent Payment Protocol (AP2)** for trusted agent payments. Already in production at Comcast (Xfinity customer support), L'Oréal (Beauty Tech agents), PayPal (agent payments), Color Health (breast cancer screening assistant), and Payhawk (financial controller).

    **Why it matters:** The "every session gets its own VM" architecture solves a real enterprise problem that container-shared agents don't (credential/state leakage between users). This is closer to what "production-ready agent infrastructure" actually needs. The multi-framework/multi-model stance is a competitive counter to Google's Gemini-only Agent Platform and Anthropic's Claude-specific surfaces. Establishes Microsoft as a third major player in enterprise agent orchestration infrastructure alongside Google Cloud and Anthropic/AWS.

- [x] **[coding]** Claude Code /ultrareview — cloud multi-agent code review that independently verifies every bug

    **What it is:** Anthropic shipped `/ultrareview`, a new Claude Code command (v2.1.86+, research preview) that launches a fleet of reviewer agents in a remote cloud sandbox. Unlike the local `/review` (single-pass, seconds), ultrareview runs a parallel multi-agent fleet that independently reproduces and verifies each finding before reporting it — so results focus on real bugs, not style suggestions. Takes ~5-10 minutes; runs in the background while you continue working. Pricing: 3 free runs through May 5, 2026 for Pro/Max; after that, ~$5-20 per review billed as extra usage. Requires Claude.ai account login (not available on Bedrock, Vertex AI, or Microsoft Foundry). Can review either the local branch diff or a GitHub PR number directly.

    **Why it matters:** The independent-verification approach directly addresses the "too many false positives" problem with AI code review. The fleet-in-cloud model is also the clearest current example of Claude Code moving from a local terminal agent toward cloud-native multi-agent workflows. Updates the Claude Code tool page and the state-of/coding picture.
