- [x] **[agents]** "Skillify" — Garry Tan's agent reliability pattern: turning failures into permanent durable skills

    **What it is:** Garry Tan published a detailed essay on the "Skillify" pattern for making AI agents reliably non-repeating on errors. Core idea: every agent failure becomes a **skill** — a markdown procedure + deterministic script + full test suite + resolver entry. The 10-step "Skillify" checklist covers: SKILL.md contract, deterministic code, unit tests, integration tests, LLM evals, resolver trigger, resolver eval, DRY audit, smoke test, and brain filing rules. The underlying architecture is called "thin harness / fat skills": the model's intelligence creates deterministic constraints that prevent the model from being wrong the same way twice. He also describes `check-resolvable` (meta-test that found 15% of skills unreachable in his setup) and the `gbrain doctor` health check toolchain. Skills are packaged as "SkillPacks" installable into any agent setup.

    **Why it matters:** This is one of the most operationally detailed published treatments of agent skill/memory architecture. It's practical and transferable — not theoretical. The "thin harness / fat skills" framing is a useful conceptual handle for the wiki's agents and workflows pages.
