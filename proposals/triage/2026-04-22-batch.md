- [ ] **[coding, security]** Slopsquatting + slopcop — LLM-hallucinated package names weaponized as supply chain attack

    **What it is:** USENIX Security 2025 study found 19.7% of LLM package recommendations reference non-existent packages across 576,000 code-generation samples. Attackers ("slopsquatting") pre-register these hallucinated names on PyPI/npm with malicious payloads that run arbitrary code via postinstall scripts. `slopcop` is a CLI tool that intercepts this: checks packages against registry age, download count, postinstall scripts, name Levenshtein distance from popular packages, and missing repo links — before anything installs. Multiple sources including Basel Ismail's "URGENT PSA" tweet and the slopcop GitHub repo.

    **Why it matters:** Concrete, empirically-grounded security threat specific to AI-assisted coding. Directly relevant to anyone using Claude Code, Codex, or Cursor. New `security/` concept page or new tool page for slopcop; update `state-of/coding` with a security note.
