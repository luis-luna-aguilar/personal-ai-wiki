# Qwen 3.6 35B-A3B

Alibaba's open-weight MoE coding model. The current signal is less "benchmark champion" than "practical threshold crossing": multiple recent sources describe it as the first local coding/agent model they would seriously use in harnesses on accessible hardware.

## Current status (as of 2026-04-21)

- 35B-parameter A3B mixture-of-experts model discussed as viable on 24GB-class hardware
- Positioned by recent commentary as a practical local-agent baseline rather than a research novelty
- Frequently mentioned alongside Hermes and other open agent stacks as part of a maturing local workflow
- Broad framing: edge / local agent workflows are getting good enough to replace proprietary models on easy and medium-difficulty tasks

## Strengths

- Local execution viability on comparatively modest hardware
- Better cost / control profile for users who want self-hosted coding agents
- Strong fit with open agent harnesses and quantized inference stacks

## Caveats

- Current claims are mostly synthesis and practitioner commentary rather than one canonical launch write-up in this batch
- Relative performance versus frontier proprietary models is still presented as directional

# Vector Lab — Qwen 3.6 as a local-agent threshold

Vector Lab's weekly newsletter frames Qwen 3.6 35B-A3B as the first open model that feels genuinely usable inside coding harnesses and agent interfaces on accessible hardware, with practical speeds and capability roughly in Sonnet-4.5 territory on some workloads.

## Key claims extracted

- Qwen 3.6 35B-A3B is described as materially better than prior dense open models for coding-agent use
- The piece emphasizes >20 tokens/sec on hardware with at least 24GB of memory
- Local use is framed as newly viable for easy and medium-difficulty agent tasks
